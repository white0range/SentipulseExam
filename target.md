## 三、项目背景

在实际工程中，核心系统往往需要支持**业务能力的动态扩展**，而不应频繁修改主程序代码。

插件化架构是一种常见解决方案。

本题要求你设计并实现一个**插件化执行系统（Plugin-based Execution System）**，主程序本身不感知具体业务逻辑，通过加载插件完成数据处理。

---

## 四、功能要求（必选）

### 1. 插件定义与规范

插件需遵循统一的接口规范，用于处理输入数据并返回结果。

示例（仅作参考，不限制）：

#### Go

```go
type Plugin interface {
    Name() string
    Version() string
    Run(data map[string]interface{}) (map[string]interface{}, error)
}
```

---

### 2. 插件加载机制

- 支持从指定目录加载插件（如 `plugins/`）
- 插件可独立开发、独立维护
- 主程序不依赖具体插件实现

---

### 3. 插件管理能力

系统需至少支持以下能力：

- 插件启用 / 禁用
- 插件运行异常不会导致主程序崩溃
- 可获取插件的基本元信息：
  - 名称
  - 版本
  - 当前状态（启用 / 禁用 / 异常）

---

### 4. 执行流程

- 主程序接收一份输入数据
- 将数据交由**已启用插件**执行
- 插件执行顺序或并发策略可自行设计
- 汇总并返回插件执行结果

---

## 五、进阶要求（加分项，非强制）

以下内容不要求全部实现，可选择性完成或仅给出设计说明：

- 插件热加载 / 热卸载机制
- 插件执行超时控制
- 插件隔离方案（如进程隔离、资源隔离思路）
- 插件依赖关系与版本约束
- 插件执行结果的失败隔离与降级策略
- 不同类型插件支持  
  （只要满足插件规范，可以支持 JS、Python、Golang 等任意语言任意形式的插件）
